Peripheral Pin Select (PPS)

Compared to other PIC32 microcontrollers, the PIC32MX220 and really the entire 1xx/2xx family, have a small number of pins. In order to get the most functionality out of these pins, peripheral pin select (PPS) is used. PPS allows the programmer to map the I/O of most digital peripherals to a selection of pins. Peripherals like SPI, UART, output compare modules (and thus PWM), input capture, external interrupts, and timer clock inputs. Note that analog pins cannot be remapped. Neither can certain digital peripherals with specialized circuitry like I2C and the SPI clock (SCK).

An example: you are using the UART1 peripheral. Instead of the UART1‚Äôs TX pin being hardwired to a specific pin as would be the case in many microcontroller‚Äôs, with PPS the programmer can choose between 8 different pins!

Admittedly, this added feature means more software configuration. Once understood, the extra work (which is actually very simple) is well worth the feature.

PPS capable pins have the designation RPn in the pinout. RP stands for ‚Äúremappable peripheral‚Äù and the n is the port and pin number. All PPS capable pins are boxed below. Note that for this package, every pin that can be a general purpose I/O pin can also use PPS.

For input pin mapping, there is a control register for each peripheral with four bits that configure what pin is being used.

For output pin mapping, there is a control register for each pin with four bits that configure what peripheral is being used.

The example below sets UART1‚Äôs RX pin (U1RX) to pin RPA4 and UART1‚Äôs TX pin (U1TX) to pin RPA0. Note that all registers and bit values are taken from the tables at the end of this tutorial. Both tables are from section 11.3 in the PIC32MX1xx/2xx datasheet.

U1RXR = 0x0002;     // configure the input U1RX to pin RPA4
RPA0R = 0x0001;     // configure the output U1TX to pin RPA0

U1RXR = 0x0002; // configure the input U1RX to pin RPA4 RPA0R = 0x0001; // configure the output U1TX to pin RPA0
