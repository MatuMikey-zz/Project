MPLAB X

MPLAB X is the free IDE that is superseding Microchip’s MPLAB. After using MPLAB for a while, I can safely say that I have been very pleased with the changes! Not only is MPLAB X cross-platform (MPLAB was windows only) but both the functionality and usability have improved.

This tutorial will be geared not to exploring all of these options but to getting set up with a PIC for the first time and learning about the basic functionality. We’ll be using the PIC32MX220F032B in the examples since M5 is currently using this chip in a DIP package for a few projects. This tutorial will also assume you have a programmer/debugger and a PIC set up for programming. If not, check out this first.

Download MPLAB X
Download MPLAB X and the lite C32 compiler, both free, from: http://ww1.microchip.com/downloads/mplab/X/index.html. Make sure to select the operating system you’ll be installing it on in the top left hand corner first.

Create A New Project

  1. File -> New Project…
  2. Microchip Embedded -> Standalone Project
  3. Find the PIC32MX220F032B chip in the drop down menu
    • Family: 32-bit MCUs (PIC32)
  4. Click on your programmer/debugger
    • we use the PICKit 3 when mentioned in tutorials
  5. Select the latest C32 compiler installed
  6. Name project and select location
    • Note that the name of the project DOES NOT have to correspond with any file names
    • Selecting ‘set as main project’ will make the project bold

Create A New File

  1. File -> New File
  2. Select your project and choose file type (C for us)
    • C Source File is blank
    • C Main file includes a standard library basic setup
  3. Name file and select location
    • extension should be ‘c’

Project Properties
Under File->Project Properties you’ll be able to see how the currently selected project is configured. Options regarding the programmer / debugger, compiler, which chip is being used, libraries, etc. are found here.

Programming
Programming is simple enough. Once your code is written, just click the “Make and Program Device (Project_Name)” icon on the top toolbar.

This will build the most recent changes done to your code as well as program the PIC with whatever programmer / debugger is set.

Debugging
The MPLAB X debugger can greatly simplify the task of finding and fixing bugs. To debug, the hardware configuration for the PICKit3 is the same as the software configuration.

For basic software debugging:

  1. Set breakpoints
    • Right-click -> toggle line breakpoint or click line number
    • Execution will stop at this point when debugger is run
  2. Create ‘Watches’
    • ‘Watches’ allow the viewing of memory location values and variables over time
    • Create one by selecting ‘New Watch’ from the right-click menu
    • They can be viewed in the ‘Variables’ or ‘Watches’ submenus

  3. Run debugger

  4. ‘Step Into’, ‘Step Through’, and ‘Continue’ between breakpoints to your hearts delight

Stopwatch
To find how long a section of code took to execute, use the ‘Stopwatch’. Can be found under Window -> Debugging -> Stopwatch. This works with both in-chip debuggers and the simulator.

If using an in-chip debugger, you’ll need to configure the stopwatches properties for when it will be active.

Common Errors
If, when attempting to program a PIC, you get the error “Target device was not found. You must connect to a target device to use PICkit 3”, although there are many possible reasons, the most common one we’ve found is that nothing is powering the PIC32. The PICKit3 has the capability to but it must first be configured. To configure, select the PICKit3 in the project properties menu. From there, select ‘power’ from the ‘Option categories’ drop down menu and check the box to apply power. Note that the default 3.25V instead of 3.3V is suitable.

Further Research
For more info about MPLAB X, check out Microchip’s MPLAB X online page or the MPLAB X IDE User’s Guide.

For a variety of help topics, the built-in online help (Help -> Help Contents) has proven useful on numerous occasions. Particularly the info under “C32 Toolchain”.
