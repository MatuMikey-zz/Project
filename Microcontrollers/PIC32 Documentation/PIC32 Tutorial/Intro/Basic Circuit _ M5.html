Basic Circuit

This tutorial will serve as a guide for setting up the hardware to program all DIP PIC32MX1xx/2xx microcontrollers with a PICKit 3 programmer/debugger on a breadboard.

QUICK SETUP
The PIC32MX1xx/2xx family is the first family of PIC32’s to have DIP microcontrollers. This means we can finally use breadboards and do all sorts of quick experimentation! First thing’s first – laying out the circuit.

NOTES
  1. All green wires are associated with the PICkit 3 programmer/debugger. The 6 headers on the left side are board-to-board headers. To connect the PICkit, always remember to put pin 1 to pin 1. That is, connect pin 1 on the PICkit (there will be an arrow on its case) to pin 1 on the PIC32 (bottom left in image). The tutorials will assume power will be provided from the PICkit. The three green wires represent the MCLR, clock (PGEC1), and data (PGED1) lines. This is all that is needed for both programming and debugging. The clock and data lines can be connected to any other PGECx/PGEDx pair as long as this is updated in the code as discussed in the ‘Configuration’ tutorial.
  2. The capacitor connected between Vcap and ground should be a low ESR (equivalent series resistance – either ceramic or tantalum) type with a value near 10uF. This is for an internal voltage regulator described at section 26.3 in the datasheet.
  3. The resistor is a pull-up resistor for the MCLR or master clear / reset pin and should be around 10k or above. See section 2.4 of the datasheet for more info.
  4. Pin VUSB should be connected to VDD when not using the USB module. If left unconnected, the PIC32 should work fine but it’s still recommended for reliability.
  5. Pin AVDD and AVSS are reference voltages for the 10 bit ADC (analog to digital converter). They should be connected even if not using the ADC.
